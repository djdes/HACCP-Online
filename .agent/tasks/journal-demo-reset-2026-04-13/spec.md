# Journal Demo Reset

## Summary
Полностью очистить демо-данные сотрудников и примеров журналов в локальной БД, затем создать новый нормальный штат с разными должностями и ровно по одному полностью заполненному примеру для каждого из 35 активных журналов.

## Acceptance Criteria
- `AC1`: отдельный reset/seed-скрипт удаляет старые демо-примеры журналов и старых сотрудников demo-организации, но сохраняет рабочий owner-логин `admin@haccp.local`.
- `AC2`: после прогона в demo-организации существует новый штат с разными `role` и `positionTitle`, а старые demo-пользователи (`chef@haccp.local`, `hotcook@haccp.local` и т.д.) больше не остаются активным составом.
- `AC3`: для всех 35 активных шаблонов существует ровно один пример журнала в БД, и у каждого примера заполнены документные данные/конфиг и есть минимум одна осмысленная запись.
- `AC4`: свежая верификация фиксирует фактические counts по сотрудникам и журналам, а `npx tsc --noEmit` проходит.

## Notes
- Область действия: текущая demo-организация, определяемая по пользователю `admin@haccp.local`.
- Сброс выполняется через отдельный скрипт, а не ленивую генерацию на страницах журналов.
