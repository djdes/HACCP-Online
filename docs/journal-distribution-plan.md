> Один большой проект разбит на три независимые фазы. Делается строго
> по порядку. Этот файл — единственный источник истины «где мы сейчас».

---

## ⚠️ Правила работы по этому плану

1. **Идём сверху вниз внутри фазы.** Не прыгаем.
2. **Пока в текущей фазе остался хоть один незавершённый пункт —
   следующую фазу не начинаем.**
3. **Завершённый пункт удаляется из файла** — не зачёркивается, а
   вырезается целиком (вместе с комментариями к нему). Это коммитится.
4. **Когда фаза становится пустой** (ни одного пункта не осталось) —
   удаляется заголовок фазы и её преамбула.
5. **Когда файл стал пустым** (нет ни одной фазы) — задача целиком
   завершена и файл `docs/journal-distribution-plan.md` удаляется.
6. **Перед стартом фазы**: все её 🔴 открытые вопросы должны иметь
   зафиксированные ответы прямо в этом файле. Если не зафиксированы —
   спросить пользователя и записать ответы перед кодом.
7. **Не открывать другие фичи** мимо этого плана пока он не пуст.
   Багфиксы и UX-полировка — можно (отдельные коммиты).

Чтобы продолжить в новом чате — просто скажи: «читай
docs/journal-distribution-plan.md и продолжай».

## Фаза 3 — Bonus + Photo Proof

> Цель: мотивационная подсистема. Сверх-работа (журналы доп. бонусные)
> оплачивается отдельно, премия идёт первому-кто-выполнил, фото-доказ
> обязательно. Превращает заполнение в гонку — поэтому делается только
> после того как Фазы 1+2 устоялись на проде.

### ✅ Зафиксированные решения (2026-04-25)

- **Q7 → per-template, default 0₽**: новое поле `JournalTemplate.bonusAmount`
  (Decimal, default 0). Премия активируется когда менеджер ставит > 0
  в `/settings/journals`. Если 0 — обычное обязательство без бонуса.
- **Q8 → страница «Премии за период» + CSV-экспорт**: новая страница
  `/dashboard/bonuses` (или `/reports/bonuses`) с фильтром по дате/
  сотруднику, выгрузка в CSV для ручного импорта в payroll. Без
  прямой интеграции с конкретным payroll-провайдером — open scope.
- **Q9 → auto-approve с EXIF-проверкой**: фото обязательно, если
  EXIF-timestamp < 5 минут от now() — `BonusEntry.status = "approved"`
  автоматически. Менеджер видит фото в фиде, может ретроактивно
  отозвать (status → "rejected") с причиной — отказывает выплату.
- **Q10 → transactional claim**: `UPDATE JournalObligation SET ...
  WHERE status='pending' AND id=$1 RETURNING` через Prisma. Кто
  первый прошёл — тому бонус. Остальным UI показывает «уже взято
  Иваном в 12:34».
